# MoAI-ADK Product Document

## 프로젝트 개요

**MoAI-ADK (MoAI Agentic Development Kit)**는 Claude Code 기반의 개발 생산성 플랫폼으로, 에이전틱 코딩을 통해 모든 개발자의 생산성과 코드 품질을 향상시키는 것을 목표로 합니다.

## 미션 & 비전

### 🎯 미션 @VISION:MISSION-001

모든 개발자가 에이전틱 코딩을 활용하여 **생산성과 코드 품질**을 동시에 향상시킬 수 있는 환경을 제공합니다.

### 🌟 비전 @VISION:STRATEGY-001

Spec-First TDD 개발 방법론을 자동화하여 개발자가 본질적인 문제 해결에 집중할 수 있도록 지원합니다.

### 🎨 핵심 가치

- **생산성 향상**: 반복 작업 자동화로 개발 속도 증대
- **품질 보장**: TRUST 5원칙 기반 체계적 품질 관리
- **학습 지원**: 개발 과정에서 자연스러운 베스트 프랙티스 습득
- **추적성 강화**: 16-Core @TAG 시스템을 통한 완전한 개발 이력 관리

## 타겟 사용자

### 🥇 Primary: 모든 개발자 @REQ:USER-001

- **개인 개발자 @REQ:PERSONA-001**: 프로젝트 품질 관리와 생산성 향상 필요
- **팀 개발자 @REQ:PERSONA-002**: 일관된 개발 프로세스와 협업 효율성 필요
- **시니어 개발자 @REQ:PERSONA-003**: 주니어 멘토링과 코드 리뷰 자동화 필요
- **DevOps 엔지니어 @REQ:PERSONA-004**: CI/CD 파이프라인과 품질 게이트 자동화 필요

### 🎯 사용자별 핵심 니즈

| 사용자 유형     | 즉시 얻고 싶은 결과            | 주요 페인 포인트                |
| --------------- | ------------------------------ | ------------------------------- |

| 팀 개발자       | 일관된 코드 품질과 리뷰 효율성 | 팀 컨벤션 준수, 협업 오버헤드   |
| 시니어 개발자   | 멘토링 자동화와 아키텍처 품질  | 반복적인 리뷰, 지식 전수 부담   |
| DevOps 엔지니어 | 안정적인 배포와 품질 보장      | 수동 검증, 장애 추적 어려움     |

## 핵심 문제 정의

### 🚨 해결하려는 주요 문제 @REQ:PROBLEM-001

1. **워크플로우 혼란 @REQ:PROBLEM-WORKFLOW-001**
   - 명세 없는 개발로 인한 요구사항 불일치
   - 테스트 부재로 인한 품질 불확실성
   - 문서와 코드 간의 동기화 부재

2. **품질 격차**
   - 개발자별 코딩 스타일과 품질 편차
   - 코드 리뷰의 일관성 부족
   - 기술 부채 누적과 추적 어려움

3. **협업 지연**
   - 컨텍스트 스위칭 비용
   - 지식 공유와 온보딩 오버헤드
   - 프로젝트 상태 파악 어려움

4. **추적성 부족**
   - 요구사항-구현-테스트 간 연결 불투명
   - 변경 이력과 의사결정 근거 부재
   - 장애 발생 시 원인 추적 어려움

### 💡 현재 실패 사례

- **명세 없는 개발**: 구현 완료 후 요구사항 변경으로 재작업
- **테스트 부재**: 프로덕션 배포 후 버그 발견으로 긴급 패치
- **문서 불일치**: README와 실제 코드 동작의 차이로 사용자 혼란
- **품질 편차**: 팀원별 다른 코딩 스타일로 유지보수 어려움

## 차별화 포인트

### 🔥 경쟁 솔루션 대비 강점

1. **자동화 심도**
   - 4단계 개발 워크플로우 완전 자동화
   - Git 작업부터 문서 동기화까지 통합 관리
   - TRUST 5원칙 기반 품질 게이트 자동 적용

2. **문서 동기화**
   - Living Document 개념으로 코드-문서 실시간 동기화
   - 16-Core @TAG 시스템을 통한 완전한 추적성
   - SPEC-Code-Test-Docs 4-way 연결

3. **추적성 보장**
   - 요구사항부터 배포까지 전체 이력 자동 기록
   - @TAG 기반 의존성 관계 시각화
   - 장애 발생 시 근본 원인 자동 추적

4. **Claude Code 네이티브**
   - AI 개발 환경에 최적화된 UX
   - 자연어 기반 개발 프로세스 가이드
   - 에이전트 기반 작업 분산 처리

### 🎯 강점 발휘 시나리오

**시나리오 1: 신규 기능 개발**

```
사용자 요청 → SPEC 자동 생성 → TDD 구현 → 문서 동기화 → PR 생성
(기존: 2-3일 → MoAI-ADK: 반나절)
```

**시나리오 2: 버그 수정**

```
이슈 접수 → 근본 원인 @TAG 추적 → 회귀 테스트 생성 → 수정 → 검증
(기존: 디버깅에 80% 시간 소요 → MoAI-ADK: 수정에 80% 시간 집중)
```

**시나리오 3: 코드 리뷰**

```
PR 생성 → TRUST 5원칙 자동 검증 → 품질 리포트 → 멘토링 가이드 제공
(기존: 리뷰어 부담 → MoAI-ADK: AI 어시스턴트 + 인간 최종 판단)
```

## 성공 지표 (KPI)

### 📊 Primary KPIs

1. **생산성 지표**
   - **개발 속도**: 기능당 평균 개발 시간 50% 단축
   - **배포 빈도**: 주 1회 → 일 1회 증가
   - **리드 타임**: 아이디어 → 배포까지 70% 단축

2. **품질 지표**
   - **버그 감소율**: 프로덕션 버그 80% 감소
   - **테스트 커버리지**: 평균 85% 이상 유지
   - **기술 부채**: 매 스프린트 10% 감소

3. **협업 지표**
   - **코드 리뷰 시간**: 평균 50% 단축
   - **온보딩 시간**: 신규 개발자 적응 기간 60% 단축
   - **컨텍스트 스위칭**: 프로젝트 간 전환 시간 40% 단축

### 📈 측정 방법

| KPI             | 베이스라인 | 목표         | 측정 주기  | 측정 방법        |
| --------------- | ---------- | ------------ | ---------- | ---------------- |
| 개발 속도       | 기능당 3일 | 기능당 1.5일 | 스프린트별 | Git 커밋 분석    |
| 버그 감소율     | 월 20건    | 월 4건       | 월별       | 이슈 트래커      |

| 리뷰 시간       | PR당 2시간 | PR당 1시간   | 주별       | GitHub Analytics |

### 🎯 단계별 성공 기준

**Phase 1: 개인 생산성 (3개월)**

- MoAI-ADK 설치 및 기본 워크플로우 적용
- 개인 프로젝트에서 50% 생산성 향상 달성

**Phase 2: 팀 협업 (6개월)**

- 팀 단위 도입 및 프로세스 표준화
- 코드 리뷰 효율성 70% 향상 달성

**Phase 3: 조직 확산 (12개월)**

- 다중 팀 환경에서 안정적 운영
- 전사 개발 품질 지표 통합 관리
